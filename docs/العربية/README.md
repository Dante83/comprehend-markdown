# comprehend-markdown

*تلمس المخطوطة وتتمتم بالتعويذة. وبعد دقائق أو ساعة (حسب ثقل المخطوطة)، ستدرك لغة Markdown بأي لسان.*

خادم MCP يقوم بترجمة ملف `README.md` الخاص بالمشروع إلى لغات أخرى، بالإضافة إلى مسار عمل (pipeline) مستقل يشغل حلقة من "الكاتب والمراجع" باستخدام نموذج LM Studio محلي.

يمكن تهيئة كل من لغتي المصدر والهدف؛ حيث أن الإنجليزية هي اللغة الافتراضية فقط. قم بتعيين `source_language` للترجمة *من* الصينية أو الإندونيسية أو أي لغة أخرى، و`target_languages` لاختيار اللغات التي سيتم التوزيع إليها (راجع [اختيار لغات المصدر والهدف](#اختيار-لغات-المصدر-والهدف)).

بالنسبة لأي مشروع مستهدف، يتوقع الخادم وجود الملفات التالية (ويقوم بإنشائها عند الحاجة):

```
<project-root>/docs/<source>/README.md  المصدر الأساسي (الإنجليزية افتراضياً)
<project-root>/docs/<lang>/README.md    النسخ المترجمة، نسخة لكل لغة
<project-root>/README.md                صفحة الهبوط لاختيار اللغة (يتم إنشاؤها تلقائياً)
```

كما يعمل النظام مع المشاريع التي لم يتم نقل ملفاتها بعد — حيث يكون المصدر لا يزال في الجذر — إذ يُستخدم ملف `README.md` الموجود في الجذر كمصدر، وفي نهاية تشغيل الـ `pipeline` يتم نقله إلى `docs/<source>/` (مثلاً `docs/English/`) ويُستبدل بصفحة هبوط قصيرة ومولدة تلقائياً تربط جميع الترجمات المتاحة.

## الإعداد (Setup)

يتم التعامل مع كل شيء (إنشاء البيئة الافتراضية venv، تثبيت وتزامن التبعيات) عبر ملف `run.sh` — لا توجد خطوة تثبيت منفصلة. يقرأ الملف الحزم من `requirements.txt` ويعيد تثبيتها عند كل تشغيل، لذا فإن جلب تبعيات جديدة يعني ببساطة تشغيله مرة أخرى.

إذا كنت ترغب في التوجيه إلى نموذج LM Studio محلي غير النموذج الافتراضي، فقم بتعديل القيم في ملف `config.json`.

### اختيار لغات المصدر والهدف

يتحكم مفتاحان في الإعدادات باتجاه الترجمة في ملف `config.json`:

- **`source_language`** — اللغة التي كُتب بها ملف `README.md` الأساسي، واسم مجلد `docs/<source_language>/`. القيمة الافتراضية هي `English`. قم بتعيينها إلى `中文` أو `Indonesia` أو أي لغة أخرى للترجمة *من* تلك اللغة بدلاً من الإنجليزية.
- **`target_languages`** — قائمة اللغات التي يترجم إليها مسار العمل (pipeline). يتم تخطي أي إدخال يتطابق مع `source_language` تلقائياً، لذا فإن ترك لغة المصدر في القائمة لا يسبب أي ضرر.

على سبيل المثال، لترجمة ملف README صيني إلى الإنجليزية والإسبانية:

```json
{
  "source_language": "中文",
  "target_languages": ["English", "Español"]
}
```

في حال حذف `target_languages` من الإعدادات، سيعود مسار العمل إلى القائمة المدمجة التي تحتوي على 18 لغة.

لا تهم قيمة `api_key` إلا إذا قمت بتفعيل خيار "Require API Key" في إعدادات خادم المطور (Developer server) في LM Studio — خلاف ذلك اتركها `"lm-studio"`، حيث يتجاهلها LM Studio. لاحظ أن هذا ينطبق فقط على وضع الـ `pipeline` وهو الجزء الوحيد هنا الذي يستدعي نقطة نهاية (endpoint) LM Studio المتوافقة مع OpenAI مباشرة؛ أما وضع الـ `serve` فلا يتحدث أبداً مع واجهة برمجة تطبيقات LM Studio لأن LM Studio *هو* عميل MCP الذي يقوم باستدعائه.

يمكن تعيين `max_tokens` هنا، ولكنني غير متأكد مما إذا كان LM Studio يحترم هذه القيمة فعلياً أم لا. تأكد من ضبط هذه القيمة لتكون 24576 على الأقل في النموذج نفسه قبل تشغيل هذا السكريبت.

## الاستخدام (Usage)

```bash
./run.sh serve    /absolute/path/to/project   # خادم MCP عبر stdio
./run.sh pipeline /absolute/path/to/project   # تشغيل main.py من البداية للنهاية
```

يتطلب كلا الوضعين مساراً **مطلقاً (absolute path)** لمجلد المشروع الذي يحتوي على ملف `README.md` المراد ترجمته. في وضع الـ `pipeline` يمكنك حذف المسار وسيُطلب منك إدخاله، بشرط أنك تقوم بالتشغيل بشكل تفاعلي عبر الطرفية (terminal):

```bash
./run.sh pipeline
Enter absolute path to project folder: /absolute/path/to/project
```

(وضع الـ `serve` لا يطلب مدخلات أبداً — بمجرد بدئه، تصبح stdin/stdout هي قناة MCP JSON-RPC نفسها، كما أن مضيفي MCP يقومون بتشغيله بشكل غير تفاعلي على أي حال.)

### `serve` — كأداة لمضيف MCP

هذا هو المسار الذي يجب أن يوجه إليه مضيف MCP (مثل LM Studio) أمر التشغيل `command` الخاص بالخادم، مع وضع المسار المطلق للمشروع المستهدف كمعامل ثابت. يوفر الخادم ما يلي:

- **tool** `write_readme(language, content)` — (كتابة ملف README للغة محددة في `docs/<language>/README.md`)
- **tool** `write_directory_readme(content)` — (كتابة ملف `README.md` الرئيسي في الجذر - صفحة اختيار اللغة)
- **resource** `docs://readme` — (المصدر: `docs/<source_language>/README.md` مع العودة إلى `README.md` في الجذر عند عدم وجوده)
- **resource** `docs://readme/{language}` — (الترجمة الموجودة، إن وجدت)
- **resource** `docs://dir_readme` — (ملف `README.md` الرئيسي في الجذر)
- **prompts** `translate_readme`, `critique_translation`, `rewrite_translation` — (حلقة الترجمة الجديدة: ترجمة، نقد، ثم إعادة كتابة)
- **prompts** `check_existing_readme`, `rewrite_from_existing_translation` — (مسار التحديث: مقارنة ترجمة موجودة بالمصدر الحالي وتصحيحها بأقل تغييرات ممكنة)
- **prompt** `create_docs_language_directory` — (بناء صفحة اختيار اللغة الرئيسية من قائمة الترجمات المتاحة)

يقوم نموذج المضيف نفسه بتوجيه استدعاءات الأدوات؛ هذا الخادم يوفر فقط عمليات الإدخال والإخراج للملفات وقوالب المطالبات (prompts).

#### إضافته إلى LM Studio

يوجد ملف إعدادات MCP الخاص بـ LM Studio في المسار `~/.lmstudio/mcp.json`. أضف إدخالاً كما يلي:

```json
{
  "mcpServers": {
    "comprehend-markdown": {
      "command": "/absolute/path/to/comprehend-markdown/run.sh",
      "args": [
        "serve",
        "/absolute/path/to/the/project/you/want/to/translate"
      ]
    }
  }
}
```

المعامل الثاني في `args` يكون ثابتاً عند الاتصال — إعدادات LM Studio هي عبارة عن JSON ثابت لا يدعم المطالبات التفاعلية، لذا يجب أن يكون المسار الفعلي للمشروع الذي تريد ترجمته، وليس مسار هذا المستودع إلا إذا كان هو المشروع المقصود بالترجمة.
