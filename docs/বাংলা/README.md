# comprehend-markdown

*আপনি সেই প্রাচীন পুঁথিতে হাত রাখলেন এবং মন্ত্রটি উচ্চারণ করলেন। কয়েক মিনিট থেকে এক ঘণ্টা পর (পুঁথির আকার অনুযায়ী), আপনি যেকোনো ভাষার মার্কডাউন বুঝতে পারবেন।*

এটি একটি MCP সার্ভার যা কোনো প্রজেক্টের `README.md`-কে অন্যান্য ভাষায় অনুবাদ করে। এর পাশাপাশি এতে একটি স্ট্যান্ডঅ্যালোন পাইপলাইন রয়েছে, যা লোকাল LM Studio মডেল ব্যবহার করে রাইটার/রিভিউয়ার লুপ (writer/reviewer loop) চালায়।

সোর্স এবং টার্গেট ল্যাঙ্গুয়েজ—উভয়টিই কনফিগার করা সম্ভব; ডিফল্ট হিসেবে ইংরেজি সেট করা আছে। অন্য কোনো ভাষা থেকে অনুবাদ করতে চাইলে `source_language` পরিবর্তন করুন, আর কোন কোন ভাষায় অনুবাদ হবে তা ঠিক করতে `target_languages` ব্যবহার করুন (বিস্তারিত দেখুন [সোর্স এবং টার্গেট ল্যাঙ্গুয়েজ নির্বাচন](#সোর্স-এবং-টার্গেট-ল্যাঙ্গুয়েজ-নির্বাচন))।

যেকোনো টার্গেট প্রজেক্টের জন্য এটি নিচের ফাইল স্ট্রাকচার প্রত্যাশা করে (প্রয়োজনে নিজেই তৈরি করে নেয়):

```
<project-root>/docs/<source>/README.md  মূল উৎস বা ক্যানোনিকাল সোর্স (ডিফল্টভাবে ইংরেজি)
<project-root>/docs/<lang>/README.md    অনুবাদিত সংস্করণ, প্রতি ভাষার জন্য একটি করে
<project-root>/README.md                ল্যাঙ্গুয়েজ-পিকার ল্যান্ডিং পেজ (অটো-জেনারেটেড)
```

যেসব প্রজেক্ট এখনও মাইগ্রেট করা হয়নি—অর্থাৎ সোর্স ফাইলটি এখনও রুট ডিরেক্টরিতে আছে—সেক্ষেত্রেও এটি কাজ করবে: রুট `README.md`-কে সোর্স হিসেবে ব্যবহার করা হবে এবং `pipeline` রান শেষ হওয়ার পর এটিকে `docs/<source>/` (যেমন: `docs/English/`) ফোল্ডারে সরিয়ে নেওয়া হবে। এরপর রুটে একটি ছোট ল্যান্ডিং পেজ তৈরি হবে যা প্রতিটি উপলব্ধ অনুবাদের লিঙ্ক প্রদান করবে।

## সেটআপ (Setup)

সবকিছু (`venv` তৈরি, ডিপেন্ডেন্সি ইনস্টল/সিঙ্ক) `run.sh`-এর মাধ্যমে সম্পন্ন হয়—আলাদা কোনো ইনস্টলেশন স্টেপ নেই। এটি `requirements.txt` থেকে প্যাকেজগুলো পড়ে এবং প্রতিবার রান করার সময় পুনরায় ইনস্টল করে, তাই নতুন ডিপেন্ডেন্সি পেতে শুধু স্ক্রিপ্টটি আবার রান করলেই হবে।

আপনি যদি ডিফল্ট মডেলের পরিবর্তে লোকাল LM Studio-র অন্য কোনো মডেল ব্যবহার করতে চান, তবে উদাহরণ কনফিগ ফাইলটি কপি করে এডিট করুন:

```bash
cp config.local.json.example config.local.json
```

`config.local.json` ফাইলটি gitignore করা থাকে এবং এটি `config.json`-এর কি (key) গুলোকে ওভাররাইড করে। ফলে আপনি মূল ডিফল্ট সেটিংস পরিবর্তন না করেই লোকালি `lm_studio_url` / `model_name` / `api_key` পরিবর্তন করতে পারবেন।

### সোর্স এবং টার্গেট ল্যাঙ্গুয়েজ নির্বাচন

দুটি কনফিগ কি-এর মাধ্যমে অনুবাদের দিক নিয়ন্ত্রণ করা হয়, এবং সার্ভার ও পাইপলাইন উভয়ই এগুলো (`config.py` থেকে) পড়ে, তাই এদের মধ্যে কোনো অমিল থাকে না:

- **`source_language`** — আপনার মূল `README.md` যে ভাষায় লেখা এবং এর জন্য ব্যবহৃত `docs/<source_language>/` ফোল্ডারের নাম। ডিফল্টভাবে এটি `"English"`। অন্য কোনো ভাষা থেকে অনুবাদ করতে চাইলে এটিকে `"中文"`, `"Bahasa Indonesia"` বা অন্য কিছু সেট করুন।
- **`target_languages`** — পাইপলাইন যে ভাষাগুলোতে অনুবাদ করবে তার তালিকা। তালিকার কোনো এন্ট্রি যদি `source_language`-এর সাথে মিলে যায়, তবে সেটি স্বয়ংক্রিয়ভাবে বাদ দেওয়া হবে; তাই তালিকায় সোর্স ল্যাঙ্গুয়েজ থাকলেও কোনো সমস্যা নেই।

উদাহরণস্বরূপ, একটি চীনা README-কে ইংরেজি এবং স্প্যানিশ ভাষায় অনুবাদ করতে:

```json
{
  "source_language": "中文",
  "target_languages": ["English", "Español"]
}
```

যদি `target_languages` উল্লেখ না করা হয়, তবে পাইপলাইনটি তার বিল্ট-ইন ১৮টি ভাষার তালিকা ব্যবহার করবে।

`api_key` শুধুমাত্র তখনই প্রয়োজন হবে যদি আপনি LM Studio-র Developer server সেটিংস থেকে "Require API Key" চালু করেন—নতুবা এটিকে `"lm-studio"` হিসেবে রেখে দিন, যা LM Studio ইগনোর করে। মনে রাখবেন, এটি শুধুমাত্র `pipeline` মোডের জন্য প্রযোজ্য, কারণ এই অংশটি সরাসরি LM Studio-র OpenAI-সামঞ্জস্যপূর্ণ এন্ডপয়েন্ট কল করে; `serve` মোড কখনোই সরাসরি LM Studio API-এর সাথে কথা বলে না, কারণ এখানে LM Studio নিজেই MCP ক্লায়েন্ট হিসেবে কাজ করে।

## ব্যবহার (Usage)

```bash
./run.sh serve    /absolute/path/to/project   # stdio MCP সার্ভার
./run.sh pipeline /absolute/path/to/project   # main.py এন্ড-টু-এন্ড রান করে
```

উভয় মোডের জন্যই প্রজেক্ট ফোল্ডারের একটি **অ্যাবসলিউট (absolute)** পাথ প্রয়োজন, যেখানে অনুবাদযোগ্য `README.md` ফাইলটি রয়েছে। টার্মিনালে ইন্টারঅ্যাক্টিভভাবে চালানোর সময় `pipeline` মোডে আপনি পাথটি বাদ দিতে পারেন, সেক্ষেত্রে এটি আপনাকে পাথ ইনপুট করতে বলবে:

```bash
./run.sh pipeline
Enter absolute path to project folder: /absolute/path/to/project
```

(`serve` মোড কখনোই প্রম্পট দেয় না—একবার শুরু হলে stdin/stdout হয়ে যায় MCP JSON-RPC চ্যানেল, এবং MCP হোস্টগুলো এটিকে নন-ইন্টারঅ্যাক্টিভভাবেই লঞ্চ করে।)

### `serve` — একটি MCP হোস্ট টুল হিসেবে

একটি MCP হোস্ট (যেমন: LM Studio) তার সার্ভার `command`-এ এই অংশটিকে পয়েন্ট করবে এবং আর্গুমেন্ট হিসেবে টার্গেট প্রজেক্টের অ্যাবসলিউট পাথ ব্যবহার করবে। এটি নিচের বিষয়গুলো এক্সপোজ করে:

- **tool** `write_readme(language, content)` — `docs/<language>/README.md` ফাইলটি লেখে।
- **tool** `write_directory_readme(content)` — রুট `README.md` (ল্যাঙ্গুয়েজ-পিকার ল্যান্ডিং পেজ) লেখে।
- **resource** `docs://readme` — সোর্স ফাইল (`docs/<source_language>/README.md`, অথবা রুট `README.md`)।
- **resource** `docs://readme/{language}` — বিদ্যমান অনুবাদ, যদি থাকে।
- **resource** `docs://dir_readme` — রুট `README.md`।
- **prompts** `translate_readme`, `critique_translation`, `rewrite_translation` — নতুন অনুবাদের লুপ।
- **prompts** `check_existing_readme`, `rewrite_from_existing_translation` — আপডেট পাথ: বিদ্যমান অনুবাদকে বর্তমান সোর্সের সাথে তুলনা করে ন্যূনতম পরিবর্তনের মাধ্যমে প্যাচ করা।
- **prompt** `create_docs_language_directory` — উপলব্ধ অনুবাদের তালিকা থেকে রুট ল্যাঙ্গুয়েজ-পিকার পেজ তৈরি করা।

হোস্টের নিজস্ব মডেল টুল কলগুলো পরিচালনা করে; এই সার্ভারটি শুধুমাত্র ফাইল I/O এবং প্রম্পট টেমপ্লেট প্রদান করে।

#### LM Studio-তে যুক্ত করা

LM Studio-র MCP কনফিগ থাকে `~/.lmstudio/mcp.json` ফাইলে। নিচের মতো একটি এন্ট্রি যোগ করুন:

```json
{
  "mcpServers": {
    "comprehend-markdown": {
      "command": "/absolute/path/to/comprehend-markdown/run.sh",
      "args": [
        "serve",
        "/absolute/path/to/the/project/you/want/to/translate"
      ]
    }
  }
}
```

দ্বিতীয় `args` এন্ট্রিটি কানেকশন টাইমে ফিক্সড থাকে—LM Studio-র কনফিগ একটি স্ট্যাটিক JSON এবং এতে ইন্টারঅ্যাক্টিভ প্রম্পট সাপোর্ট নেই। তাই এখানে আপনার অনুবাদযোগ্য প্রজেক্টের প্রকৃত পাথ দিতে হবে, এই রিপোজিটরির পাথ নয় (যদি না আপনি এই রিপোজিটরি নিজেই অনুবাদ করতে চান)।

### `pipeline` — স্ট্যান্ডঅ্যালোন, কোনো MCP হোস্ট ছাড়াই

এটি কনফিগার করা `target_languages`-এর জন্য সম্পূর্ণ translate $\rightarrow$ critique $\rightarrow$ revise লুপটি নিজে পরিচালনা করে (ডিফল্ট ১৮টি ভাষা: Deutsch, Español, Français, Italiano, Polski, Português, Русский, Tiếng Việt, ไทย, 中文, 日本語, 한국어, العربية, हिन्दी, বাংলা, Bahasa Indonesia, اردو, Naijá — তালিকা বা সোর্স ল্যাঙ্গুয়েজ পরিবর্তন করতে [সোর্স এবং টার্গেট ল্যাঙ্গুয়েজ নির্বাচন](#সোর্স-এবং-টার্গেট-ল্যাঙ্গুয়েজ-নির্বাচন) দেখুন)। এটি রাইটার/রিভিউয়ার টার্নগুলোর জন্য সরাসরি LM Studio-র OpenAI-সামঞ্জস্যপূর্ণ এন্ডপয়েন্ট কল করে এবং ফাইল I/O করার জন্য stdio-র মাধ্যমে `server.py`-এর একটি ইন্টারনাল কপি স্পন (spawn) করে। LM Studio UI ব্যবহার না করে ব্যাচ-ট্রান্সলেশনের জন্য এটি অত্যন্ত কার্যকর। লোকাল মডেলে রানটাইম ডকুমেন্টের আকারের ওপর নির্ভর করে—ছোট README-র জন্য কয়েক মিনিট, আর বড় খণ্ড করা ডকুমেন্টের জন্য এক ঘণ্টা পর্যন্ত সময় লাগতে পারে।

যেসব ভাষার জন্য ইতিমধ্যে `docs/<language>/README.md` আছে, সেগুলো একদম শুরু থেকে অনুবাদ করা হয় না: রিভিউয়ার বিদ্যমান অনুবাদটিকে বর্তমান সোর্সের সাথে তুলনা করেন এবং শুধুমাত্র কিছু পরিবর্তন থাকলে রাইটার ন্যূনতম এডিটের মাধ্যমে তা প্যাচ করেন। আপ-টু-ডেট অনুবাদগুলো সম্পূর্ণ এড়িয়ে যাওয়া হয়, ফলে README এডিট করার পর পাইপলাইন পুনরায় চালালে শুধু পুরনো অংশগুলোই আপডেট হবে।

প্রতিটি ভাষার কাজ শেষ হওয়ার পর, রানটি সম্পন্ন হয় রুট-লেভেলের সোর্স `README.md`-কে (প্রয়োজনে) `docs/<source_language>/`-এ সরিয়ে নিয়ে এবং রুট `README.md`-কে একটি ছোট ল্যাঙ্গুয়েজ-পিকার পেজ হিসেবে পুনরায় তৈরি করে।

এর জন্য LM Studio-র লোকাল সার্ভার চালু থাকা প্রয়োজন (দেখুন `config.json`) এবং যেকোনো চ্যাট মডেল লোড করা থাকতে হবে—পাইপলাইনটি প্লেইন টেক্সট কমপ্লিশনের মাধ্যমে মডেলটিকে পরিচালনা করে এবং ফাইল রাইটিং নিজেই করে, তাই মডেলটির OpenAI-স্টাইল টুল কল সাপোর্ট করার **প্রয়োজন নেই** (নিচে "Large-README handling" দেখুন)। তবে পুরো তালিকাটি চালানোর আগে একটি ভাষায় টেস্ট করে নেওয়া বুদ্ধিমানের কাজ হবে।

### Large-README handling (বড় README ব্যবস্থাপনা)

একটি বড় README-কে (যেমন `A-Starry-Sky` / `a-restless-ocean` ডকসগুলো প্রায় ৩৫–৬০ KB) একবারেই অনুবাদ করা লোকাল মডেলের ক্ষেত্রে প্রধান ঝুঁকি: এটি আউটপুট টোকেন বা কনটেক্সট শেষ করে ফেলে এবং শেষের অংশটি নিঃশব্দে কেটে যায় (truncate)। `pipeline` মোডটি এই সমস্যা এড়ানোর জন্যই তৈরি করা হয়েছে:

- **ডিজাইনের কারণেই টুল-মুক্ত।** মডেলটি শুধুমাত্র প্লেইন টেক্সট তৈরি করে—প্রতিটি অনুবাদ, রিরাইট এবং ডিরেক্টরি পেজ তার রিপ্লাই হিসেবে আসে, এবং অর্কেস্ট্রেটর সার্ভারের `write_readme` / `write_directory_readme` টুলের মাধ্যমে তা সেভ করে। মডেলটিকে কোনো টুল-কল আর্গুমেন্টের ভেতর পুরো ডকুমেন্ট ঢোকাতে বলা হয় না। লোকাল *reasoning* মডেলে (যেমন Gemma 4) এই পথটি আর্গুমেন্ট JSON-কে ট্রানকেট করে ফেলে এবং এন্ডপয়েন্ট থেকে `peg-gemma4 format` / malformed-output এরর আসে; টেক্সট রিটার্ন করার মাধ্যমে এই সমস্যা পুরোপুরি এড়িয়ে যাওয়া হয়।
- **`max_tokens`** (ডিফল্ট `32768`) প্রতিটি কমপ্লিশনের সময় স্পষ্টভাবে পাঠানো হয়, যাতে LM Studio-র ছোট ডিফল্ট লিমিটে আটকে না গিয়ে পুরো সেকশন বা ড্রাফট শেষ হতে পারে। যদি কোনো কমপ্লিশন তবুও `length`-এর কারণে থেমে যায়, তবে পাইপলাইন সতর্ক করে দেয় যাতে আপনি এটি বাড়িয়ে নিতে পারেন।
- **সেকশন চাঙ্কিং (Section chunking)** — যখন সোর্স ফাইলটি `chunk_threshold_chars` (ডিফল্ট `12000`) অতিক্রম করে এবং `chunk_translation` এর মান `true` থাকে, তখন সোর্সটিকে টপ-লেভেল (`## `) মার্কডাউন হেডিং অনুযায়ী ভাগ করা হয় (এটি fence-aware, তাই কোড ব্লকের ভেতরের `##`-কে স্পর্শ করে না)। প্রতিটি সেকশন আলাদা কমপ্লিশনে অনুবাদ করা হয় এবং অর্কেস্ট্রেটর অংশগুলো জোড়া লাগিয়ে সেভ করে। ফলে কোনো একক মডেল কলকে পুরো ডকুমেন্ট আউটপুট করতে হয় না।
- **প্রতি-সেকশন রিভিউ (Per-section review)** — যখন `review_sections` এর মান `true` থাকে (ডিফল্ট), প্রতিটি সেকশনের জন্য একই critique $\rightarrow$ revise লুপ চলে, তবে তা শুধুমাত্র ওই নির্দিষ্ট সেকশনের জন্য সীমাবদ্ধ থাকে: রিভিউয়ার অনুবাদিত সেকশনটিকে তার সোর্সের সাথে তুলনা করেন এবং রাইটার সংশোধন করেন যতক্ষণ না রিভিউয়ার সম্মত হন (অথবা `MAX_ITERATIONS` পূর্ণ হয়)। যেহেতু একটি সেকশন ছোট হয়, তাই রিভিউ এবং রিরাইট ট্রানকেশন সিলিং-এর অনেক নিচে থাকে, যা বড় README-র পুরো ডকুমেন্টের রিভিউকে নিরাপদ করে তোলে।

চাঙ্কিং পদ্ধতিতে ইচ্ছাকৃতভাবেই পরে দ্বিতীয়বার পুরো ডকুমেন্টের রিভিউ চালানো হয় না—কারণ পুরো বড় README-টি একবারেই আউটপুট করতে গেলে পুনরায় ট্রানকেশন সমস্যা দেখা দেবে, এবং প্রতি-সেকশন পাস ইতিমধ্যে তা কভার করেছে। কোনো রিভিউ ছাড়াই বড় ডক অনুবাদ করতে `review_sections` কে `false` করুন, অথবা অরিজিনাল সিঙ্গেল-শট বিহেভিয়ার ফোর্স করতে `chunk_translation` কে `false` করুন (যা পুরো ডকুমেন্টের রিভিউ লুপটি চালাবে)। থ্রেশহোল্ডের নিচের সিঙ্গেল-শট পাথ অপরিবর্তিত থাকে।
