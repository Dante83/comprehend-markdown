# comprehend-markdown

*আপনি সেই প্রাচীন পুঁথিতে হাত রাখলেন এবং মন্ত্রটি উচ্চারণ করলেন। কয়েক মিনিট থেকে এক ঘণ্টা পর (পুঁথির গুরুত্ব অনুযায়ী), আপনি যেকোনো ভাষার মার্কডাউন বুঝতে পারবেন।*

এটি একটি MCP সার্ভার যা কোনো প্রজেক্টের `README.md`-কে অন্যান্য ভাষায় অনুবাদ করে। এর পাশাপাশি এতে একটি স্ট্যান্ডঅ্যালোন পাইপলাইন রয়েছে, যা লোকাল LM Studio মডেল ব্যবহার করে রাইটার/রিভিউয়ার লুপ (writer/reviewer loop) পরিচালনা করে।

সোর্স এবং টার্গেট ভাষা—উভয়টিই কনফিগার করা সম্ভব; ডিফল্ট হিসেবে ইংরেজি সেট করা আছে। অন্য কোনো ভাষা থেকে অনুবাদ করতে চাইলে `source_language` পরিবর্তন করুন (যেমন: চীনা, ইন্দোনেশীয় বা অন্য কিছু), এবং কোন কোন ভাষায় অনুবাদ হবে তা নির্ধারণ করতে `target_languages` ব্যবহার করুন (বিস্তারিত দেখুন [Choosing source and target languages](#choosing-source-and-target-languages))।

যেকোনো টার্গেট প্রজেক্টের জন্য এটি নিচের ফাইলগুলো প্রত্যাশা করে (এবং প্রয়োজন অনুযায়ী তৈরি করে):

```
<project-root>/docs/<source>/README.md  মূল উৎস বা ক্যানোনিকাল সোর্স (ডিফল্টভাবে ইংরেজি)
<project-root>/docs/<lang>/README.md    অনুবাদিত সংস্করণ, প্রতিটি ভাষার জন্য একটি করে
<project-root>/README.md                ল্যাঙ্গুয়েজ-পিকার ল্যান্ডিং পেজ (অটো-জেনারেটেড)
```

যেসব প্রজেক্ট এখনো মাইগ্রেট করা হয়নি—অর্থাৎ সোর্স ফাইলটি এখনো রুট ডিরেক্টরিতে আছে—সেক্ষেত্রেও এটি কাজ করবে: রুট `README.md`-কে সোর্স হিসেবে ব্যবহার করা হবে এবং `pipeline` রান শেষ হওয়ার পর এটিকে `docs/<source>/` (যেমন: `docs/English/`) ফোল্ডারে সরিয়ে নেওয়া হবে। এরপর রুটে একটি ছোট ল্যান্ডিং পেজ তৈরি হবে যা প্রতিটি উপলব্ধ অনুবাদের লিঙ্ক প্রদান করবে।

## Setup

সবকিছু (venv তৈরি, ডিপেন্ডেন্সি ইনস্টল/সিঙ্ক) `run.sh`-এর মাধ্যমে সম্পন্ন হয়—আলাদা কোনো ইনস্টলেশন স্টেপের প্রয়োজন নেই। এটি `requirements.txt` থেকে প্যাকেজগুলো পড়ে এবং প্রতিবার রান করার সময় পুনরায় ইনস্টল করে, তাই নতুন ডিপেন্ডেন্সি যুক্ত করতে হলে শুধু স্ক্রিপ্টটি আবার রান করলেই হবে।

আপনি যদি ডিফল্ট মডেলের পরিবর্তে লোকাল LM Studio-র অন্য কোনো মডেল ব্যবহার করতে চান, তবে `config.json`-এর ভ্যালুগুলো পরিবর্তন করুন।

### Choosing source and target languages

`config.json`-এ দুটি কনফিগ কি (key) অনুবাদের দিক নিয়ন্ত্রণ করে:

- **`source_language`** — আপনার মূল `README.md` যে ভাষায় লেখা এবং এর `docs/<source_language>/` ফোল্ডারের নাম। ডিফল্টভাবে এটি `English` থাকে। অন্য কোনো ভাষা থেকে অনুবাদ করতে চাইলে এখানে `中文`, `Indonesia` বা অন্য কিছু লিখুন।
- **`target_languages`** — পাইপলাইন যেসব ভাষায় অনুবাদ করবে তার তালিকা। তালিকার কোনো এন্ট্রি যদি `source_language`-এর সাথে মিলে যায়, তবে সেটি স্বয়ংক্রিয়ভাবে বাদ দেওয়া হবে; তাই তালিকায় সোর্স ল্যাঙ্গুয়েজ থাকলেও কোনো সমস্যা নেই।

উদাহরণস্বরূপ, একটি চীনা README-কে ইংরেজি এবং স্প্যানিশ ভাষায় অনুবাদ করতে চাইলে:

```json
{
  "source_language": "中文",
  "target_languages": ["English", "Español"]
}
```

যদি `target_languages` উল্লেখ না করা হয়, তবে পাইপলাইনটি তার বিল্ট-ইন ১৮টি ভাষার তালিকা ব্যবহার করবে।

LM Studio-র Developer server সেটিংসে যদি "Require API Key" চালু করা থাকে, তবেই কেবল `api_key`-এর প্রয়োজন হবে—নতুবা এটিকে `"lm-studio"` হিসেবে রেখে দিন, যা LM Studio উপেক্ষা করে। মনে রাখবেন, এটি শুধুমাত্র `pipeline` মোডের জন্য প্রযোজ্য, কারণ এই অংশটি সরাসরি LM Studio-র OpenAI-কম্প্যাটিবল এন্ডপয়েন্ট কল করে; `serve` মোড কখনোই সরাসরি LM Studio API-এর সাথে কথা বলে না, কারণ এখানে LM Studio নিজেই MCP ক্লায়েন্ট হিসেবে কাজ করে।

এখানে `max_tokens` সেট করার অপশন আছে, তবে LM Studio এটি সঠিকভাবে মেনে চলে কি না তা নিশ্চিত নয়। এই স্ক্রিপ্টটি চালানোর আগে মডেলের সেটিংসে গিয়ে নিশ্চিত করুন যে এর ভ্যালু অন্তত ২৪৫৭৬ (24576) দেওয়া আছে।

## Usage

```bash
./run.sh serve    /absolute/path/to/project   # stdio MCP server
./run.sh pipeline /absolute/path/to/project   # runs main.py end-to-end
```

উভয় মোডের জন্যই যে প্রজেক্ট ফোল্ডারে `README.md` রয়েছে, তার একটি **অ্যাবসোলুট পাথ (absolute path)** প্রয়োজন। টার্মিনালে ইন্টারঅ্যাক্টিভভাবে চালানোর সময় `pipeline` মোডে আপনি পাথটি বাদ দিতে পারেন, সেক্ষেত্রে এটি আপনাকে পাথ ইনপুট করার জন্য প্রম্পট করবে:

```bash
./run.sh pipeline
Enter absolute path to project folder: /absolute/path/to/project
```

(`serve` মোডে কোনো প্রম্পট আসে না—একবার শুরু হলে stdin/stdout হয়ে যায় MCP JSON-RPC চ্যানেল, এবং MCP হোস্টগুলো এটিকে নন-ইন্টারঅ্যাক্টিভভাবেই লঞ্চ করে।)

### `serve` — as an MCP host tool

একটি MCP হোস্ট (যেমন: LM Studio) তার সার্ভার `command`-এ এই পাথটি সেট করবে এবং আর্গুমেন্ট হিসেবে টার্গেট প্রজেক্টের অ্যাবসোলুট পাথ প্রদান করবে। এটি নিচের বিষয়গুলো এক্সপোজ করে:

- **tool** `write_readme(language, content)` — `docs/<language>/README.md` লেখে
- **tool** `write_directory_readme(content)` — রুট `README.md` (ল্যাঙ্গুয়েজ-পিকার ল্যান্ডিং পেজ) লেখে
- **resource** `docs://readme` — সোর্স ফাইল (`docs/<source_language>/README.md`, অথবা রুট `README.md`)
- **resource** `docs://readme/{language}` — বিদ্যমান অনুবাদ (যদি থাকে)
- **resource** `docs://dir_readme` — রুট `README.md`
- **prompts** `translate_readme`, `critique_translation`, `rewrite_translation` — নতুন অনুবাদের লুপ
- **prompts** `check_existing_readme`, `rewrite_from_existing_translation` — আপডেট পাথ: বিদ্যমান অনুবাদকে বর্তমান সোর্সের সাথে তুলনা করে ন্যূনতম পরিবর্তনের মাধ্যমে প্যাচ করা
- **prompt** `create_docs_language_directory` — উপলব্ধ অনুবাদের তালিকা থেকে রুট ল্যাঙ্গুয়েজ-পিকার পেজ তৈরি করা

হোস্টের নিজস্ব মডেল টুল কলগুলো পরিচালনা করে; এই সার্ভারটি শুধুমাত্র ফাইল I/O এবং প্রম্পট টেমপ্লেট প্রদান করে।

#### Adding it to LM Studio

LM Studio-র MCP কনফিগ ফাইলটি থাকে `~/.lmstudio/mcp.json`-এ। সেখানে নিচের মতো একটি এন্ট্রি যোগ করুন:

```json
{
  "mcpServers": {
    "comprehend-markdown": {
      "command": "/absolute/path/to/comprehend-markdown/run.sh",
      "args": [
        "serve",
        "/absolute/path/to/the/project/you/want/to/translate"
      ]
    }
  }
}
```

দ্বিতীয় `args` এন্ট্রিটি কানেকশন টাইমে ফিক্সড থাকে—LM Studio-র কনফিগ একটি স্ট্যাটিক JSON এবং এতে কোনো ইন্টারঅ্যাক্টিভ প্রম্পট সাপোর্ট নেই। তাই এখানে আপনাকে সেই প্রজেক্টের প্রকৃত পাথ দিতে হবে যা আপনি অনুবাদ করতে চান, এই রিপোজিটরির নিজস্ব পাথ নয় (যদি না আপনি এই রিপোজিটরিটিই অনুবাদ করতে চান)।
